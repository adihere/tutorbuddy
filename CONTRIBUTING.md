## Adding New Screenshots

1. Take screenshots following guidelines in `screenshots/README.md`
2. Place them in the `screenshots/` directory
3. Update references in main README.md if adding new images
4. Ensure all screenshots reflect current UI

## Documentation Standards

- Use clear, concise language appropriate for both technical and non-technical users
- Include code examples with proper formatting and comments
- Reference actual code files with line numbers when possible
- Follow the established markdown structure and formatting
- Test all links and references before submitting

## Making Changes

1. Fork the repository
2. Create a feature branch for your documentation changes
3. Make your changes following the established structure
4. Test all changes locally
5. Submit a pull request with clear description of changes

## Review Process

All documentation changes will be reviewed for:
- Accuracy and completeness
- Consistency with existing style
- Proper formatting and structure
- Working links and references
